Importable NPC AI
Dev Tool

For a quick rundown of what has been accomplished, please watch the video above.

This is a developer tool meant for creating complex NPC behaviour in a pub/restaurant setting. By configuring the settings, You can change the maximum number of NPCs present, the frequency they perform their actions, and how quickly they leave the premises. They have multiple personality traits which manipulate their behaviours, for example alcohol increases confidence and confidence is for instance used to determine where an NPC will sit near people. The tool also provides access to the environment's appearance and each object's affect on the NPCs.

I created 2 example scenes to assist in getting familiar with the system and a simple set-up prefab, ready to import into a scene. This portfolio piece features:

  • Queueing system
  • Beverage and gender preferences
  • Sequence of possible behaviours for the NPC to perform
  • Various world objects like bins, toilets and tables
  • Bar and food stands
  • NPC spawner and exit
  • Debug console that shows current NPC state and data
  • Configurable prefab appearance for NPC and in game objects (e.g. the way the food appears)
  • Cinfigurable personality traits and behaviours (e.g. queue spacing, max size and time taken to collect

Procedural City Generator

A developer tool that procedurally creates a city. It's configurable with the following attributes:

  • Width
  • Length
  • Road percentage
  • Small to big house ratio
  • Filler house ratio

It additionally consists of the following systems:

  • Multiple menus
  • Controllable camera player replicating Unity controls
  • Street decorations, such as benches or lamps
  • Buildings of different, colours, heights and widths, with some being toggled to include a balcony
  • An importable prefab
  • A complicated tile based road system (visible above)

It works by firstly generating a road system and then checks empty land plots to potentially generate buildings. These use a series of logic functions, 2d arrays and raycasts to determine if any should be generated. If the developer so chooses then they can configure the appearance of everything that is generated. This tool was solely designed by me and didnt follow any online tutorials.

First Person Shooter

A game inspired by the original Doom and Wolfenstein 3D. It features:

  • 2 weapons
  • 3 levels
  • Minor and boss enemies with configurable AI behaviour, e.g. wether it runs away or takes cover
  • Ammunition, armour and keycard pickups
  • Controllable player
  • All encompassing audio effects
  • Multiple menus
  • Level select
  • Control screen with configurable mouse sensitivity

The enemies can be selected to perform certain behaviours like patrolling, covering or attacking. They utilise things like raycast and dot product checks to confirm vision but will also toggle to check for the player when a weapon is fired within hearing range. Stuff like rate of fire can also be configured.

The player can find rocket ammunition which lets them use their rocket launcher. On collision the rockets will explode and damage or destroy everything in range. This includes destructible objects like certain walls or door power generators. The levels are designed to allow for multiple playstyles but will punish poor game play.

the player can pause at any time. The controls are explained inside the controls screen and the mouse sensitivity can be changed at any time in case it feels too high or low. You can also choose to exit the game or load a certain level.

Projector, CCTV, Mirror
and Decal Effects

This project was used to create multiple effects ready for use by a designer. These are:

  • Video projector with audio support
  • Realtime CCTV projector
  • Still image projector
  • Decal projector
  • Mirror object that uses camera to show reflection

The first 3 can have their shapes, brightness and other settings configured. The projectors are designed to allow easy implementation into a level to increase player immersion. Therefore settings like projectable distance, brightness and light shape among others are configurable.
